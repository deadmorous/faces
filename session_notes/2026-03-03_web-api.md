# 2026-03-03 — Web API implementation

## Goal

Add a FastAPI-based HTTP layer to the faces tool so that interactive
workflows (classify, label, browse) can be driven from a future web UI
rather than the sequential CLI prompt.  Scan stays CLI-only.

## What was built

### `faces/algo.py` — shared algorithm module

Extracted the core ML logic that was previously embedded in the CLI
command functions so that both the CLI and the new API can call it
without duplication:

- `classify_candidates(db, threshold, min_size, since, until)` — runs
  the single-linkage min-dist classification, returns groups of
  unlabeled faces matched to named people plus a list of unmatched faces
- `run_clusterize(db, threshold, reset)` — runs agglomerative
  clustering with must-link / cannot-link constraints, stores results,
  returns summary stats; raises `ValueError("clusters_exist")` when
  clusters already exist and `reset=False`

### `faces/web/` — FastAPI package

```
faces/web/
  __init__.py
  main.py       FastAPI app; lifespan opens DB once into app.state
  deps.py       get_cfg() / get_db() Depends helpers
  models.py     all Pydantic request/response schemas
  routers/
    __init__.py
    images.py   /img/*
    clusters.py /api/clusters
    people.py   /api/people
    photos.py   /api/photos
    faces.py    /api/faces
    classify.py /api/classify
    clusterize.py /api/clusterize
```

## Endpoints (13 total)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/img/photo/{md5}` | Stream original JPEG from disk |
| GET | `/img/face` | Dynamically crop a face and return as JPEG |
| GET | `/api/clusters` | List clusters, sorted by size desc |
| GET | `/api/clusters/{id}` | Cluster detail with all faces and links |
| PATCH | `/api/clusters/{id}` | Rename cluster; optionally stick label |
| GET | `/api/people` | List all named people (distinct sticky names) |
| GET | `/api/people/{name}` | Photos containing a person |
| GET | `/api/photos` | Paginated photo list |
| GET | `/api/photos/{md5}` | Photo detail with all detected faces |
| PATCH | `/api/faces/{md5}/{bbox}` | Set or clear sticky label on one face |
| GET | `/api/classify/candidates` | Unlabeled faces grouped by predicted person |
| POST | `/api/classify/labels` | Bulk submit labels for any number of faces |
| POST | `/api/clusterize` | Trigger agglomerative clustering |

Notes:
- `bbox` in `/api/faces/{md5}/{bbox}` is dash-separated (`x1-y1-x2-y2`)
  to avoid URL-encoding issues.
- `/img/face` query params: `md5`, `bbox` (comma-separated), `padding`
  (default 0.25), `size` (default 224).
- `/api/clusterize` returns `409 Conflict` if clusters exist and
  `reset` is false.

## Design decisions

- **DB opened once** in the FastAPI lifespan and stored in `app.state`;
  `get_db` / `get_cfg` Depends helpers read from there — no per-request
  reconnect overhead.
- **No auth** — personal single-user tool.
- **Sync endpoints** — acceptable at this collection size; the
  clustering loop is CPU-bound and runs in the request thread.
- **`faces/algo.py`** keeps the CLI commands thin: they now delegate
  to the same functions the API uses, so behaviour stays consistent.
- `img_url` values embedded in every face response follow the pattern
  `/img/face?md5=X&bbox=x1,y1,x2,y2` so a UI can render thumbnails
  without any extra API calls.

## How to run

```bash
uvicorn faces.web.main:app --reload
# OpenAPI docs at http://localhost:8000/docs
```
